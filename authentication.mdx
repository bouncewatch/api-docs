## API Key Authentication

All API requests must include your API key in the request header.

```bash
X-API-Key: your_api_key_here
```

<CodeGroup>
```bash cURL
curl -X GET "https://api.bouncewatch.com/v1/company/stripe.com" \
  -H "X-API-Key: bw_live_1234567890abcdef" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch('https://api.bouncewatch.com/v1/company/stripe.com', {
  headers: {
    'X-API-Key': 'bw_live_1234567890abcdef',
    'Content-Type': 'application/json'
  }
});
```

```python Python
import requests

response = requests.get(
    'https://api.bouncewatch.com/v1/company/stripe.com',
    headers={
        'X-API-Key': 'bw_live_1234567890abcdef',
        'Content-Type': 'application/json'
    }
)
```

```php PHP
$ch = curl_init('https://api.bouncewatch.com/v1/company/stripe.com');
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'X-API-Key: bw_live_1234567890abcdef',
    'Content-Type: application/json'
]);
$response = curl_exec($ch);
$data = json_decode($response, true);
```
</CodeGroup>

## Getting Your API Key

<Steps>
  <Step title="Sign In">
    Go to [BounceWatch Dashboard](https://bouncewatch.com/api-panel/api-keys)
  </Step>
  <Step title="Copy Your API Key">
    Your API key is displayed on the dashboard. Copy it to use in your requests.
  </Step>
</Steps>

## Security Best Practices

<CardGroup cols={2}>
  <Card title="Do's" icon="check" color="#22c55e">
    - Store API keys in environment variables
    - Use HTTPS for all requests
    - Rotate keys periodically
    - Use different keys for different environments
    - Monitor usage for anomalies
  </Card>
  <Card title="Don'ts" icon="xmark" color="#ef4444">
    - Commit API keys to version control
    - Share keys publicly or in logs
    - Use keys in client-side JavaScript
    - Reuse keys across environments
    - Share keys via email or chat
  </Card>
</CardGroup>

## Environment Variables

Store your API key securely using environment variables:

<Tabs>
  <Tab title="Node.js">
    ```bash .env
    BOUNCEWATCH_API_KEY=bw_live_1234567890abcdef
    ```

    ```javascript app.js
    require('dotenv').config();
    const API_KEY = process.env.BOUNCEWATCH_API_KEY;
    ```
  </Tab>
  <Tab title="Python">
    ```bash .env
    BOUNCEWATCH_API_KEY=bw_live_1234567890abcdef
    ```

    ```python app.py
    import os
    from dotenv import load_dotenv

    load_dotenv()
    API_KEY = os.getenv('BOUNCEWATCH_API_KEY')
    ```
  </Tab>
  <Tab title="PHP">
    ```bash .env
    BOUNCEWATCH_API_KEY=bw_live_1234567890abcdef
    ```

    ```php config.php
    <?php
    $dotenv = Dotenv\Dotenv::createImmutable(__DIR__);
    $dotenv->load();
    
    $apiKey = $_ENV['BOUNCEWATCH_API_KEY'];
    ```
  </Tab>
  <Tab title="Shell">
    ```bash
    export BOUNCEWATCH_API_KEY=bw_live_1234567890abcdef
    ```

    ```bash
    curl -H "X-API-Key: $BOUNCEWATCH_API_KEY" \
      https://api.bouncewatch.com/v1/company/stripe.com
    ```
  </Tab>
</Tabs>

## Authentication Errors

| Error Code | Message | Solution |
|------------|---------|----------|
| `401` | `Invalid API key` | Check that your API key is correct |
| `401` | `API key missing` | Add the `X-API-Key` header |
| `401` | `API key expired` | Generate a new API key |
| `403` | `API key disabled` | Contact support to reactivate |

```json Error Response
{
  "success": false,
  "error": "unauthorized",
  "message": "Invalid API key provided"
}
```

## Key Rotation

For enhanced security, you can regenerate your API key periodically:

<Steps>
  <Step title="Regenerate Key">
    Click "Regenerate" in the dashboard to create a new API key. This will immediately invalidate your current key.
  </Step>
  <Step title="Update Your Application">
    Update your environment variables with the new key.
  </Step>
  <Step title="Test">
    Verify your application works with the new key.
  </Step>
</Steps>

<Warning>
  Regenerating your API key will immediately invalidate the old one. Make sure to update all your applications before regenerating.
</Warning>
