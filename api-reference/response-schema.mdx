## Overview

This page documents the complete response structure for all BounceWatch API endpoints. Use this as a reference to understand exactly what data you'll receive.

<Info>
  All responses include a `success` boolean and follow a consistent structure. Data is returned in the `data` object.
</Info>

## Response Wrapper

Every API response follows this structure:

```json
{
  "success": true,
  "mode": "cached",
  "data": {
    // Module data here
  },
  "credits_used": 26,
  "credits_breakdown": {
    "base": 10,
    "enrichments": {
      "funding": 10,
      "team": 6
    }
  },
  "modules_included": ["base", "funding", "team"],
  "from_recent_enrichment": false,
  "from_recent_cache": false,
  "credits_remaining": 4974
}
```

<Info>
  If `from_recent_enrichment` or `from_recent_cache` is `true`, you're receiving cached data from a request made within the last 24 hours - no additional credits are charged.
</Info>

### Base Company Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Company display name |
| `description` | string | Company description/about |
| `founded_year` | integer | Year the company was founded |
| `company_status` | string | Operational status: `active`, `acquired`, `closed` |
| `phone` | string | Primary phone number |
| `email` | string | Primary contact email |
| `domain` | string | Company website domain |
| `linkedin_url` | string | LinkedIn company page URL |

| `headquarter_city` | string | HQ city name |
| `registered_address` | string | Official registered address |
| `office_locations` | array | List of other office locations |
| `employee_count` | integer | Exact employee count (if available) |
| `employee_range` | string | Employee range bucket |
| `legal_name` | string | Official legal entity name |
| `company_number` | string | Company registration number |
| `company_type` | string | Legal entity type |
| `date_of_creation` | string | Company registration date |
| `jurisdiction` | string | Legal jurisdiction |
| `sic_codes` | array | Industry classification codes |

## Error Responses

<Info>
  **Note:** If a domain is not found in cached mode, the API now **automatically triggers realtime enrichment** (202 Accepted) instead of returning a 404 error. The examples below apply to other error scenarios.
</Info>

```json
{
  "success": false,
  "error": "insufficient_credits",
  "message": "Not enough credits for this request",
  "credits_needed": 20,
  "credits_available": 5
}
```

| Error | HTTP Code | Description |
|-------|-----------|-------------|
| `insufficient_credits` | 402 | Not enough credits |
| `invalid_api_key` | 401 | API key is invalid or missing |
| `rate_limit_exceeded` | 429 | Too many requests |
| `invalid_module` | 400 | Invalid enrichment module specified |
| `webhook_required` | 400 | Webhook URL required for new domain enrichment |
